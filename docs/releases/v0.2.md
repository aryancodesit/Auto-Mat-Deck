# v0.2 — OTP Pairing & Action Editor

**Release date:** 2026-07-12
**Version:** `amd-desktop 0.2.0`

## Summary

Two major features: **OTP-based pairing** replaces hidden tray-approval with an explicit, secure credential flow, and **action-specific editor forms** replace raw JSON editing with typed forms for each action type.

## What's new

### OTP Pairing (EP-001, EP-004)

- Desktop generates a 6-digit OTP + QR code from the Pairing tab.
- Android enters the code in the spike app → Desktop validates via `PairingManager`.
- Sessions are one-time use, expire after 5 minutes, and can be cancelled explicitly.
- Legacy tray-approval fallback retained when `pair_request` has no `pairing_code` field.
- New module: `apps/desktop/src/pairing.rs` — `PairingManager`, `PairingSessionSnapshot`, `ValidationResult`.

### Action Editor (EP-002, EP-004)

- Per-action-type editor forms: **Launch** (app path + Browse), **Open URL** (URL input), **Open File** (file path + Choose File), **Lock** (no settings), **Notify** (title + body).
- Payloads are constructed from typed EditorUi drafts, not raw JSON input.
- Native file dialogs via `rfd` crate.

## Protocol changes

| Change | Detail |
|--------|--------|
| `pair_request` payload | Added optional `pairing_code` field (6-digit string). When present, agent validates against `PairingManager` instead of prompting tray. |
| `pair_rejected` payload | Added `reason` field with stable machine-readable codes: `code_mismatch`, `expired`, `cancelled`, `already_used`, `no_session`, `user_declined`, `timeout`. Android maps codes to display text locally. |

See `docs/architecture/protocol.md` for full message catalog.
