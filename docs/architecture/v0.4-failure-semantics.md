# v0.4 — Failure Semantics

**Status:** Draft
**Date:** 2026-07-14

Normative references: sprint-3-failure-semantics.md, ADR-020, v0.4-scope.md.

---

## 1. Failure taxonomy

| # | Failure Mode | Layer | Scope | Behaviour | Recovery |
|---|-------------|-------|-------|-----------|----------|
| 1 | `RuntimeProjection` → DTO mapping fails (unexpected field) | Publisher | Projection thread | Panic → fatal process failure | Process restart |
| 2 | Serialization failure (`serde_json::to_string` returns `Err`) | Publisher | Projection thread | Log warning, drop projection, channel unchanged | Next publish() call |
| 3 | `watch::Sender::send_replace()` called with no receivers | Watch channel | Projection thread | Succeeds — value retained in channel | Value delivered when receiver subscribes |
| 4 | `watch` channel closed (sender dropped during shutdown) | Watch channel | Connection task (Tokio) | `Receiver::changed()` returns `Err(RecvError::Closed)` → connection task breaks loop, closes WebSocket | Process shutting down — expected |
| 5 | Receiver lags (multiple values sent between reads) | Watch channel | Connection task (Tokio) | `Receiver::changed()` resolves to `true` once; `borrow()` returns latest value only | Automatic — consumer converges to latest |
| 6 | WebSocket write failure | Connection task | Tokio runtime | Log connection-scoped warning → terminate connection handler → receiver drops → sink drops | Reconnect on new WebSocket connection |
| 7 | WebSocket read failure or peer disconnect | Connection task | Tokio runtime | `read.next()` returns `None` or `Err` → loop exits → receiver dropped | Reconnect on new connection |
| 8 | Android disconnect | Connection task | Tokio runtime | `read.next()` returns `None` → loop exits → receiver dropped | Reconnect on new connection |
| 9 | Android receives malformed `active_profile_state` | Android parse | Android spike | Log warning, do not update retained state, keep connection alive | Next valid message restores correctness |
| 10 | Android receives unsupported `schema_version` | Android parse | Android spike | Log warning, do not update retained state, keep connection alive | N/A — client must be updated for new schema |
| 11 | Projection publication during shutdown race | Publisher | Projection thread | `watch::Sender::send_replace()` is infallible in Tokio 1.x — no panic path exists under the ownership model (Sender owned by publisher, alive during `publish()`) | Delivery loss is accepted; process is shutting down |
| 12 | Multiple write failures across all connections | Publisher + connections | Combined | Each connection handles independently; no cross-connection propagation | Per-connection recovery |

## 2. Failure boundaries

```
DesktopRuntime
      │
      │  (unchanged from Sprint 3)
      ▼
ProjectionEngine
      │
      ▼
PublicationPolicy
      │
      ▼
ProjectionTransportPublisher
      │
      ├── DTO mapping:     panic → fatal process failure
      ├── serialization:   log + drop → channel unchanged
      │
      ▼
watch::Sender
      │
      ├── no receivers:    store value, no delivery
      ├── sender dropped:  closed channel, tasks exit
      │
      ▼
watch::Receiver → Connection task
      │
      ├── channel closed:  exit loop (shutdown)
      ├── write failure:   log → terminate handler
      ├── read failure:    exit loop (disconnect)
      │
      ▼
WebSocket sink
      │
      ▼
Android client
      │
      ├── malformed JSON:  log, ignore, stay connected
      ├── unsupported version: log, ignore, stay connected
```

**Rule:** Failures at or below the `ProjectionTransportPublisher` (including
watch channel, connection tasks, WebSocket I/O) never propagate into
`DesktopRuntime`, `ProjectionEngine`, `PublicationPolicy`, or the observer
thread. The runtime continues operating.

## 3. Recovery policy

| Failure | Recovery |
|---------|----------|
| Serialization failure | None (next `publish()` call may succeed) |
| No receivers | Value retained — instant delivery when receiver subscribes |
| Channel closed during shutdown | None (process is exiting) |
| Receiver lag | Automatic — consumer converges to latest value |
| WebSocket write failure | Connection task exits — reconnect creates new task |
| Android disconnect | Reconnect on new WebSocket connection |
| Malformed Android message | Stay connected — next valid message restores state |
| Unsupported schema_version | Stay connected — client must update |

**No component implements retry logic, back-off, or circuit breaking.
The lossy policy makes retry unnecessary.**

## 4. Observable effects by deployment phase

| Phase | Failure scenario | Observable effect |
|-------|-----------------|-------------------|
| Development | Mock transport publisher panics | Test fails with clear panic message |
| Development | Missing serialization field | Compile error (DTO is typed) |
| Production | WebSocket disconnect | Android shows stale state until reconnect |
| Production | WebSocket reconnect | Android receives latest `active_profile_state` |
| Production | Serialization failure | One state update lost; next update restores correctness |
| Production | Android receives unsupported schema | Android retains previous state; desktop unaffected |
