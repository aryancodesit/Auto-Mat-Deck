## 2. Desktop derivation failures

| # | Failure | Behaviour | CSS channel |
|---|---------|-----------|-------------|
| 24 | active_profile_id = None | Derive valid no-surface null triple → publish | Updated |
| 25 | active_profile_id = Some(id), Profile(id) not found in Document | DerivationFailure → log, drop | Unchanged |
| 26 | Document access error | DerivationFailure → log, drop | Unchanged |
| 27 | Serialization (serde) failure | Log, drop | Unchanged |

## 3. Transport failures

| # | Failure | Behaviour | CSS channel |
|---|---------|-----------|-------------|
| 28 | CSS sender send_replace with no CSS receivers | Value retained | Unchanged (delivered when subscribes) |
| 29 | CSS channel sender dropped (shutdown) | CSS receiver observes closed | CSS removed from select! |
| 30 | CSS receiver lag | changed() coalesces | Latest only |
| 31 | WebSocket write failure | Connection task exits, both receivers dropped | Reconnect restores |
| 32 | Channel A closed, Channel B alive | Remove A branch from select!, continue B | CSS forwarding continues |

## 4. Recovery policy

| Failure | Recovery |
|---------|----------|
| Any parse/rejection | Stay connected, retain last valid CSS |
| Association mismatch | CSS retained but ineligible. Next matching publish restores eligibility |
| APS null or CSS null triple | No CSS eligible until non-null APS with matching CSS |
| Derivation failure on Desktop | Next Document/Profile change retries derivation |
| Serialization failure | Next publish attempt |
| Channel closed (single) | Remove branch, continue |
| Both channels closed | Connection task exits (shutdown) |
| WebSocket disconnect | Reconnect restores latest from both channels |

## 5. Failure boundaries

```
Desktop [derivation]
  ├── DerivationFailure:     log, channel unchanged
  ├── SerializationFailure:  log, channel unchanged

Desktop → Watch ChB → Receiver → Connection task → WebSocket → Mobile
                                                              │
                                                    Android parser
                                                      ├── invalid: log, retain previous
                                                      └── valid + mismatch: retain, ineligible
```

All CSS failures are isolated from APS. APS failures are isolated from CSS.
Connection-task failures affect both (WebSocket disconnect), as expected.

## 6. Compatibility with v0.4

All v0.4 failure modes unchanged. CSS adds new failure paths that are
independent.
