## 2. Connection + Trusted Handshake

```mermaid
sequenceDiagram
    participant M as Mobile
    participant D as Desktop

    M->>D: WebSocket connect
    M->>D: identify
    D-->>M: trusted
    Note over D: CTx clones APS Receiver + CSS Receiver
    D-->>M: active_profile_state (latest retained)
    Note over D,M: CSS sent after APS (implementation choice)
    D-->>M: control_surface_state (latest retained)
    Note over M: APS valid → CSS valid, profile_id matches → ELIGIBLE
```

The implementation MAY send APS then CSS sequentially for deterministic
display. Mobile correctness does NOT depend on this ordering.

## 4. Active Profile Transition — APS arrives before CSS

```mermaid
sequenceDiagram
    participant PE as ProjectionEngine
    participant M as Mobile

    Note over PE, M: Active Profile: gaming → coding

    PE-->>M: active_profile_state(profile_id=coding)
    Note over M: retained APS = coding
    Note over M: retained CSS = coding? No, CSS is previous gaming
    Note over M: eligible = NO (CSS gaming, APS coding: mismatch)

    PE-->>M: control_surface_state(profile_id=coding, pages, profile_name=Coding)
    Note over M: retained CSS = coding, profile_id matches APS coding
    Note over M: eligible = YES
```

Safe convergence. Temporary ineligibility is architecturally valid.

---

## 5. Active Profile Transition — CSS arrives before APS

```mermaid
sequenceDiagram
    participant PE as ProjectionEngine
    participant M as Mobile

    Note over PE, M: Active Profile: gaming → coding

    PE-->>M: control_surface_state(profile_id=coding, pages, profile_name=Coding)
    Note over M: retained CSS = coding (valid parse, retained)
    Note over M: retained APS = gaming (previous still valid)
    Note over M: eligible = NO (APS gaming, CSS coding: mismatch)

    PE-->>M: active_profile_state(profile_id=coding)
    Note over M: now retained APS = coding, matches CSS coding
    Note over M: eligible = YES
```

CSS arrival before APS is architecturally valid. CSS retained with full
state, merely ineligible until APS converges.

## 12. Association Mismatch Retained but Ineligible

```mermaid
sequenceDiagram
    participant M as Mobile

    Note over M: APS = gaming
    M-->>M: receive CSS coding

    Note over M: 1. Parse CSS (valid)
    Note over M: 2. Retain CSS = coding, pages, profile_name
    Note over M: 3. Derive eligibility
    Note over M: APS gaming ≠ CSS coding → INELIGIBLE

    Note over M: CSS coding RETAINED as latest valid state
    Note over M: CSS coding NOT discarded
```
