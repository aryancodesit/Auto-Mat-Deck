## 7. Shutdown Sequence

```
Main Thread        Projection Thread      Agent Server (Tokio)      Observer Thread
      │                    │                       │                      │
      │── signal shutdown ─┼───────────────────────┤                      │
      │                    │                       │                      │
      │                    │── (may still be        │                      │
      │                    │    calling publish())  │                      │
      │                    │                       │── server loop exit    │
      │                    │                       │── Tokio runtime drops │
      │                    │                       │   (connection tasks   │
      │                    │                       │    abort, sinks drop) │
      │                    │                       │                      │
      │                    │── loop exit            │                      │
      │                    │── drop publisher       │                      │
      │                    │   (Sender dropped)    │                      │
      │                    │── thread exit          │                      │
      │                    │                       │                      │
      │── join projection   │                       │                      │
      │── join server       │                       │                      │
      │                    │                       │                      │
      │                    │                       │                      │── observer exit
      │── join observer ───┼───────────────────────────────────────────→│
      │                    │                       │                      │
      │── main returns     │                       │                      │
```

### Shutdown invariants

1. `watch::Sender` lifetime is independent of the Tokio runtime. Either the
   server runtime or the projection thread may terminate first during shutdown.
2. If the Tokio runtime terminates first, connection tasks (and their
   `watch::Receiver` clones) are aborted. The projection thread may briefly
   continue calling `send_replace()` — safe because `send_replace()` is
   infallible, runtime-independent, and valid with zero receivers.
3. If the projection thread exits first, the `watch::Sender` is dropped and
   any remaining receivers observe a closed channel — they terminate cleanly.
4. Delivery loss during process shutdown is accepted under ADR-015.
5. **Join order is non-normative for v0.4.** Main-thread join ordering does
   not establish thread exit ordering or runtime lifetime coordination.
   No additional shutdown acknowledgement, lifecycle barrier, or coordination
   primitive is required.
