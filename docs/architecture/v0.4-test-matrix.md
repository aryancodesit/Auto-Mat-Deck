## 1. Background

The Sprint 3 test matrix defines 29 architectural and implementation scenarios.
21 projection-layer implementation tests exist (covering ProjectionEngine,
PublicationPolicy, TransitionCell, and integration with MockPublisher).

v0.4 adds transport-layer tests. All 21 existing projection-layer tests must
continue to pass without modification.

## 2. Architectural Invariant Tests (v0.4 additions)

These tests verify properties that must hold regardless of concrete
transport implementation.

| # | Test | Invariant | Type |
|---|------|-----------|------|
| 30 | `ProjectionTransportPublisher` implements `ProjectionPublisher` | Static type check — trait conformance | Compile-time |
| 31 | OS-thread synchronous publish | `publish()` returns without awaiting — no runtime required | Architectural |
| 32 | No unbounded projection queue | Transport never accumulates historical projections; only latest retained | Architectural |
| 33 | No WebSocket sink in projection thread | Projection thread never accesses socket I/O | Architectural (review) |
| 34 | No runtime types in transport code | Transport crate does not import `DesktopRuntime`, `ProfileRuntime` | Compile-time |
| 35 | No Win32 in transport code | Same as Sprint 3 test #8 | Compile-time |

### 3.6 Shutdown

| # | Test | Behaviour | Notes |
|---|------|-----------|-------|
| 59 | Sender lifetime independent of Tokio runtime | `send_replace()` infallible with zero receivers, no runtime required | Unit (architectural proof) |
